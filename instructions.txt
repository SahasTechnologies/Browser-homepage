What's an API
Right now you may be wondering, "What IS an API?". API stands for Application Programming Interface. APIs are the root of almost all apps and websites. For example, when programming for Android and IOS, you use an API to communicate with the devices firmware and ask for things such as battery life or location. In our case, we will be using REST APIs. REST (Representational State Transfer) is the most widely used API architecture for web services. To use a REST API, you simply send a request to an endpoint and it will respond with a JSON response.

Examples of different uses for REST APIs:

Weather forecast
Photos from space
Cat facts
JSON Example:

{
  "firstName": "John",
  "lastName": "Doe",
  "age": 30,
  "isEmployee": true
}

Now that we have a function with our API URL, we need to call it. When calling APIs or doing anything that has a chance of not working, we use try and catch blocks. These allow the computer to try something, and if an error occurs, then it will run a different set of code. This is crucial to preventing errors that crash your website.

Within this try block, we first have to use the fetch function, this will call the API and wait for a response. Once we get a response, we will check the ok property of the response. This property is a boolean (true or false) that checks if the response was successful (response code within the 200 range). If it was a successful response, we need to do one final check to make sure the item in the url property is a image. If all that is true, then we return the url element of the json.

Put this inside the getBackground function:

try {
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`Response status: ${response.status}`);
  }

  const result = await response.json();
  console.log(result);

  if (result.media_type !== "image") {
    console.log("APOD returned a non-image media type.");
    return null;
  }

  return result.url;
} catch (error) {
  console.log(error.message);
  return null;
}
Now that we have a function that returns the url of the picture, we will create a function that runs on load. Then, we will set the background-image css property of the div we made.

Tip!
In JS, you can use backticks along with ${} to dynamically add the value of variables to strings. Ex: let value="good"; console.log(Cats are ${value}!);

window.onload = function () {
  getBackground().then(function (imageUrl) {
    if (!imageUrl) return;

    console.log(imageUrl);

    if (background) {
      background.style["background-image"] = `url('${imageUrl}')`;
    }
  });
};
That should be it for the background!

Now that we have an element to put the time in, we need to write code to figure out the current time. For this next section, I'm going to give you all the things you need to know, and you should try to code the time updating yourself!

JavaScript has a built in object called Date. From this object, you can get things like the time. (Remember, the time is from when you created the object, not current, so you want to make a new object every time you update the time). When you run dateObject.getTime() it returns the time in Unix time. Unix time is the number of milliseconds since Jan 1st, 1970. JavaScript has functions that will automatically give you the hour, min, sec, etc, but that's not learning, so my challenge for you is to write functions to convert the Unix time into human readable time.

Here are some things you should know to make clock work:

How to run code every x seconds
To run code every x secs, you need to use the setInterval function. Here is an example:

setInterval(() => {
  // code goes here
}, 1000); // <-- this is the interval. In this case, 1000 ms or 1 sec.
How to use date object
To use the built in date object you need to do a few things. First, you need to create an instance of the object every time you want to check the time or date. Like this: let dateObject = new Date();

Then, you will need to read the time. dateObject.getTime(); returns Unix time as a number.

To account for time zones, you will need to use the dateObject.getTimezoneOffset(); function. This returns the difference between UTC (what getTime() returns) and the users computer time in minutes. If this is acting weird remember that difference is UTC - Local, and that Unix time is in milliseconds, not minutes (1000 ms / s).

Remainder
One very helpful piece of math is finding the remainder. In JavaScript that is done with the % symbol. Ex: 10 % 3 = 1. You could find the remainder of the number total millis to the number of them in a day to find millis in current day.

Floor
You may want to round down, for example if you use division to calc the time and you get something like 9.45:54.5. You can use Math.floor(); to round the number down every time.

Logging data to console
If you want to log stuff to the console for debugging reasons, you can use the console.log(); function. Ex: console.log(Some logs, and a ${variable}!);

Editing a paragraph
Now that you can log the time, you might want to actually change it on the site, to do this you can use the .innerText attribute of your time element. Ex:

const time = document.getElementById("time");

time.innerText = `Some nice text. Here's a ${variable}!`;
That should be everything you need to know to make the time work. YOU GOT THIS, LOCK IN!!

One thing you may be asking is how do we get the location of the user. Luckily for us, JavaScript has a function just for that. navigator.geolocation.getCurrentPosition(); This function takes one parameter, which is a function for it to call once it finds the location. That function should also have one parameter that takes the geolocationPosition. Inside that function you can get the longitude and latitude like this: geolocationPosition.coords.latitude and geolocationPosition.coords.longitude.
